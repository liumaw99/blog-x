# Docker 高频问答笔记

这一篇按问答形式整理 Docker 常见面试题，适合临时复习与快速回忆。

## 1. 什么是 Docker？

Docker 是一个容器化平台，用来将应用及其运行环境一起打包、分发和运行。

它的核心价值是：

- 环境一致
- 部署方便
- 启动快
- 资源开销相对虚拟机更小

## 2. Docker 和虚拟机有什么区别？

### Docker

- 共享宿主机内核
- 启动快
- 镜像和容器更轻量
- 资源占用更少

### 虚拟机

- 每台虚拟机有完整操作系统
- 隔离更完整
- 启动更慢
- 资源开销更大

一句话概括：

Docker 更轻，虚拟机更重但更完整。

## 3. Docker 的核心概念有哪些？

最常见的四个：

- 镜像 `image`
- 容器 `container`
- 仓库 `registry`
- 数据卷 `volume`

### 镜像是什么？

镜像是一个只读模板，里面包含应用运行需要的文件和环境。

### 容器是什么？

容器是镜像运行后的实例。

### 仓库是什么？

仓库是存放镜像的地方，例如 Docker Hub。

## 4. 镜像和容器的区别是什么？

- 镜像是静态模板。
- 容器是镜像启动后的运行实例。

可以理解为：

- 镜像像类模板
- 容器像对象实例

## 5. Docker 常用命令有哪些？

### 镜像相关

```bash
docker images
docker pull nginx
docker rmi nginx
docker build -t myapp .
```

### 容器相关

```bash
docker ps
docker ps -a
docker run -d --name nginx -p 8080:80 nginx
docker stop nginx
docker start nginx
docker rm nginx
docker exec -it nginx /bin/sh
```

### 日志与排错

```bash
docker logs nginx
docker inspect nginx
docker stats
```

## 6. `docker run` 做了什么？

`docker run` 本质上做了几件事：

1. 检查本地是否有指定镜像。
2. 没有的话先拉取镜像。
3. 基于镜像创建容器。
4. 分配文件系统、网络、可写层。
5. 执行启动命令。

## 7. `docker run` 和 `docker start` 有什么区别？

- `docker run` 是“创建并启动”一个新容器。
- `docker start` 是“启动一个已经存在的容器”。

## 8. `CMD` 和 `ENTRYPOINT` 的区别是什么？

### `CMD`

- 用来指定默认启动命令或默认参数。
- 可以被 `docker run` 后面的命令覆盖。

### `ENTRYPOINT`

- 用来指定固定主命令。
- 更适合定义容器必须执行的程序。

## 9. `RUN` 和 `CMD` 的区别是什么？

- `RUN` 在构建镜像时执行。
- `CMD` 在容器启动时执行。

## 10. `COPY` 和 `ADD` 的区别是什么？

- `COPY` 只负责复制文件。
- `ADD` 除了复制，还能自动解压本地压缩包，甚至支持 URL。

面试回答建议：

- 开发中优先使用 `COPY`
- 只有确实需要额外功能时再用 `ADD`

## 11. 什么是数据卷？

数据卷是 Docker 提供的持久化存储机制，用来让容器数据脱离容器本身存在。

作用：

- 持久化数据
- 共享数据
- 便于备份和迁移

## 12. 为什么需要数据卷？

因为容器删除后，容器可写层中的数据通常也会消失。

而数据库、上传文件、配置文件等数据不能跟着容器一起没掉，所以需要挂载到卷或宿主机目录。

## 13. 什么是端口映射？

端口映射就是把宿主机端口转发到容器内部端口。

例如：

```bash
docker run -p 8080:80 nginx
```

含义是：

- 访问宿主机 `8080`
- 实际转发到容器 `80`

## 14. `EXPOSE` 和 `-p` 有什么区别？

- `EXPOSE` 是声明容器会使用哪个端口。
- `-p` 才是真正把端口映射到宿主机。

## 15. Docker 网络有哪几种常见模式？

常见有：

- `bridge`
- `host`
- `none`

### `bridge`

默认模式，容器挂到桥接网络上。

### `host`

容器直接使用宿主机网络。

### `none`

容器不配置网络。

## 16. 容器之间如何通信？

若多个容器处于同一个 Docker 网络中，它们可以通过：

- 容器 IP
- 容器名
- 服务名

通常更推荐使用服务名或容器名，而不是写死 IP。

## 17. 什么是镜像分层？

Docker 镜像是分层存储的。

每一条会生成文件变更的指令，往往都会形成一层。这样做的好处是：

- 节省存储
- 提高复用率
- 构建时可以利用缓存

## 18. 如何减小镜像体积？

常见方法：

1. 选更小的基础镜像，例如 `alpine`。
2. 合并 `RUN` 命令，减少层数。
3. 使用多阶段构建。
4. 配置 `.dockerignore`。
5. 不要把构建缓存、源码垃圾文件打进镜像。

## 19. 什么是多阶段构建？

多阶段构建就是在一个 `Dockerfile` 中使用多个 `FROM`。

常见场景：

- 第一阶段编译代码
- 第二阶段只复制编译产物到运行镜像

优点：

- 镜像更小
- 更安全
- 最终镜像更干净

## 20. Docker Compose 是做什么的？

Docker Compose 用来定义和管理多个容器服务。

适合：

- 应用 + 数据库
- 应用 + Redis
- 多服务本地开发环境

## 21. `docker compose up` 和 `docker compose down` 分别做什么？

- `docker compose up -d`：启动服务
- `docker compose down`：停止并删除相关容器和网络

如果加 `-v`：

```bash
docker compose down -v
```

还会删除相关数据卷。

## 22. 容器删除后数据一定会丢失吗？

不一定。

分两种情况：

- 数据在容器可写层：容器删了，数据大概率没了。
- 数据挂载到数据卷或宿主机目录：容器删了，数据还在。

## 23. 为什么容器里不推荐跑多个主进程？

Docker 设计理念通常是：

- 一个容器一个主要职责
- 一个容器一个主进程

这样做更利于：

- 扩展
- 运维
- 日志管理
- 故障隔离

## 24. Docker 常见排错命令有哪些？

```bash
docker ps -a
docker logs 容器名
docker inspect 容器名
docker exec -it 容器名 /bin/sh
docker stats
docker network ls
docker volume ls
```

排错顺序建议：

1. 先看容器是否启动成功。
2. 再看日志。
3. 再看配置和环境变量。
4. 必要时进入容器内部排查。

## 25. Docker 的优点和局限分别是什么？

### 优点

- 部署快
- 环境一致
- 易迁移
- 资源占用较低
- 适合微服务和 CI/CD

### 局限

- 隔离性不如完整虚拟机
- 网络、存储概念对新手不够直观
- 生产环境仍需配合监控、编排、安全治理

## 26. 面试中的高频总结答案

若被问“Docker 的核心价值是什么”，可以答：

```text
Docker 通过容器技术把应用及其依赖环境统一打包，解决了环境不一致、部署麻烦、扩缩容低效等问题。
```

若被问“镜像、容器、仓库分别是什么”，可以答：

```text
镜像是静态模板，容器是镜像运行后的实例，仓库是存放和分发镜像的地方。
```

若被问“为什么 Docker 比虚拟机轻量”，可以答：

```text
因为 Docker 容器共享宿主机内核，不需要为每个实例都启动完整操作系统，所以启动更快、资源占用更低。
```

## 27. 小结

主公若要临场速记，只需先抓住这几句话：

- Docker 是轻量级容器化平台。
- 镜像是模板，容器是实例。
- `RUN` 构建时执行，`CMD` 启动时执行。
- `COPY` 更常用，`ADD` 功能更多。
- `-p` 做端口映射，`-v` 做数据持久化。
- 同一网络内，容器通常可以按名字互相访问。
- Compose 用来管理多个服务。
